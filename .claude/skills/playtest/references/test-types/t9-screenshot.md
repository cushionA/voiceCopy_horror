# T9: スクリーンショット回帰テスト — ビジュアルエビデンスの専門家

## 役割
他テスト（T2-T8）の検証ポイントを読み、目視確認が必要なタイミングでのキャプチャ計画を設計する。

## テスト設計プロセス

### 入力（Step 2 DESIGNの他テスト設計から受け取る情報）
- T2: 入力操作の重要タイミング（攻撃ヒット、コンボ遷移等）
- T4: アニメーション遷移の検証ポイント
- T8: UI表示切替の検証ポイント
- T7: エッジケース操作の前後

### 設計手順
1. **他テストの検証ポイントから「目視が有効な瞬間」を抽出**
2. **エフェクト・アニメーション系**: ヒット/遷移の瞬間にキャプチャ
3. **UI系**: 表示切替の前後にキャプチャ
4. **エラー系**: 予期しない状態が発生した場合の自動キャプチャ
5. **最低限の2枚**: テスト開始前（ベースライン）+ 完了後（最終状態）

### 設計品質基準
- 他テストのFail時に原因推定に役立つキャプチャがある
- 無駄な大量撮影はしない（検証ポイントに紐づいたものだけ）

## 実行手段
- `/unicli`: Screenshot.Capture
- 保存先: `Assets/Screenshots/` ディレクトリ

## 対象
- UI画面の見た目（HUD、メニュー、ダイアログ）
- エフェクト表示（攻撃エフェクト、被弾エフェクト）
- アニメーション特定フレーム（攻撃モーション、死亡モーション）
- ステージビジュアル（背景、タイルマップ）
- バグ再現のエビデンス

## キャプチャタイミング

| タイミング | 用途 |
|-----------|------|
| テスト開始前 | ベースライン |
| 攻撃ヒット時 | エフェクト・ポップアップ確認 |
| UI操作後 | メニュー表示・遷移確認 |
| エラー発生時 | バグエビデンス |
| テスト完了後 | 最終状態の記録 |

## 実行パターン

### 基本キャプチャ
```bash
unicli exec Screenshot.Capture
```

### 特定タイミングでのキャプチャ（AutoInputと組み合わせ）
1. AutoInput（T2）で攻撃入力
2. 数フレーム待ち（ヒットエフェクト表示まで）
3. Screenshot.Capture
4. 次のアクションへ

### エラー時の自動キャプチャ
- ANALYZE_ERRORS でエラー検出時に自動撮影
- エラーレポートにスクリーンショットパスを添付

## 命名規則
Screenshot.Capture が自動的にタイムスタンプ付きファイルを生成。
レポートにはファイルパスを記載する。

## 設計指針
- 全テスト実行時に最低限「開始前」「完了後」の2枚は撮る
- エラー時は必ず撮る（再現困難なバグのエビデンス）
- 大量撮影はディスク容量に注意（定期的にクリーンアップ推奨）
- PlayMode中のみ有効（EditModeではGameViewが更新されない）

## 結果判定
- T9単体でPass/Failは判定しない
- 他テスト（T2-T8）の補助エビデンスとして機能
- レポートの「Evidence」セクションにパスを記載
