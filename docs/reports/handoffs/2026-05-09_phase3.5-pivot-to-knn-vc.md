## 結論先行

**CosyVoice3 路線は撤退決定**。Phase 3.5 で判明したアーキテクチャ齟齬
(DiT mu に encoder hidden state が必要、token encoder の ONNX export
未着手で残作業 4-5 日以上) と、voice_horror の用途 (少女幽霊 1 体固定の声)
に対する **過剰なモデル規模 (合計 2.6GB VRAM)** を踏まえ、
**kNN-VC + WavLM 路線へ pivot** する (Phase 5)。

kNN-VC は zero-shot で学習不要、WavLM は **VC と類似度判定 (BAD/GOOD ED 分岐) を
1 モデル 2 役で兼ねられる** ため、voice_horror の narrative
(録音蓄積 → 徐々に同化) と原理的にも適合する。

## 撤退判断の根拠

### 技術的根拠 (Phase 3.5)

`docs/reports/handoffs/2026-05-07_phase3.5-noise-and-length.md` 詳細:

| 問題 | 状態 |
|------|------|
| DiT mu 入力アーキテクチャ齟齬 | 解決には token encoder + length_regulator の ONNX export が必要 (未着手、2-3日相当) |
| DiT 出力 mel 58% -inf 発散 | mu 直しても CFG / cosine scheduler / 数値スケーリングで再発の可能性 |
| 長さ問題 A (T=100 切断) | hift-only chunk loop で実装的解決済 (RunHiftOnly) |
| MelExtractor matcha 一致 | Hann window symmetric / n_fft=1920 padding 未解決 |

### 戦略的根拠

| 観点 | CosyVoice3 | kNN-VC + WavLM |
|------|-----------|----------------|
| 残作業 | encoder export + 互換性 + 統合 = 4-5日 | local spike + Sentis port = 3-5日 |
| 合計 VRAM | ~2.6GB (4 モデル) | ~900MB (2 モデル兼用) |
| アーキテクチャ複雑度 | LLM token decoder + DiT + flow + vocoder | WavLM + kNN + HiFiGAN |
| Sentis 互換リスク | 高 (Phase 3.5 で実証) | 低 (transformer SSL は枯れている) |
| narrative 適合 | 単発 zero-shot 変換 | プール蓄積で「徐々に混ざる」演出が原理的に一致 |
| 類似度判定 | 別途モデル要 (ECAPA-TDNN 等) | WavLM で兼用可 (VRAM 節約) |
| 学習要件 | zero-shot 完成済 | zero-shot (matching set 構築は forward のみ) |

### 品質判断

`https://bshall.github.io/knn-vc/` で公式デモを聴感確認:
- ターゲット 5 分音声で「かなり似てる」品質 (ユーザー判定)
- ホラー演出基準には十分
- whisper conversion bonus も voice_horror のささやき演出に直結

## 注意点・引き継ぎ

### kNN-VC 路線の前提

- **学習は不要、特徴抽出 (forward) のみ**で matching set 構築
- プレイヤー声蓄積は 1 回 forward のみ (~0.1-0.3 秒/3 秒録音)
- WavLM (frozen) を VC と類似度判定で兼用
- ベクトル空間を厚くするセリフ設計 (ATR503 + ホラー特有発話) で 5 分音声でも十分

### CosyVoice3 路線の保全

- Phase 4 ブランチに撤退時点スナップショットを残してある
- `voiceCoppy_test/legacy_cosyvoice/README.md` に再開手順を記載
- 将来 kNN-VC が NG だった場合 or 品質を更に求める場合は token encoder ルートで再開可能
- ただし VRAM 圧迫問題は残る → 慎重判断
