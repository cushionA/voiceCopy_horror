# UI設計リクエストフォーム

UI画面を作成する際、以下の項目を埋めて `/create-ui` に渡す。
未記入項目がある場合はSkillが対話で補完する。

## フォーマット

```
# UIリクエスト: [画面名]

## 基本情報（必須）
- 画面種別: [HUD | メニュー | ダイアログ | インベントリ | 設定 | カスタム]
- 目的: [この画面で何をするか、1-2文]

## コンテンツ（必須）
- 表示要素:
  - [要素1]: [説明]
  - [要素2]: [説明]
- 操作要素:
  - [要素1]: [種類(ボタン/スライダー/トグル等)] [動作説明]
  - [要素2]: [種類] [動作説明]

## 操作（必須）
- 入力方式: [マウス+キーボード | ゲームパッド | 両対応]
- フォーカスナビゲーション: [上下選択 | グリッド | タブ切替 | なし]

## 見た目（推奨）
- カラースキーム: [ダーク | ライト | 半透明オーバーレイ | カスタム]
- アクセントカラー: [色名 or HEXコード]
- 背景: [半透明黒 | ぼかし風 | 全画面不透明 | なし(HUD)]
- 密度: [コンパクト | 通常 | ゆったり]

## アニメーション（推奨）
- レベル: [なし | 最小限(遷移のみ) | 標準(フィードバック含む) | リッチ]
- 具体的な要件:
  - [例: バーの増減にTweenアニメーション]
  - [例: 画面表示時にフェードイン]

## 画面遷移（推奨）
- この画面への入り方: [自動表示 | ボタン押下 | イベントトリガー]
- この画面からの出方: [ESCで閉じる | ボタンで遷移 | 自動消滅]
- 戻り先: [前の画面 | 特定画面名 | なし]

## データバインディング（該当する場合）
- [UI要素名] ← [データソース.プロパティ]
- [UI要素名] ← [データソース.プロパティ]

## 参考（任意）
- 参考ゲーム/画像: [ゲーム名やスクリーンショットパス]
- 既存UI参照: [プロジェクト内の既存UIを基にする場合]
- 備考: [その他の要望]
```

## 記入例

```
# UIリクエスト: ポーズメニュー

## 基本情報（必須）
- 画面種別: メニュー
- 目的: ゲーム中にESCで開き、再開/設定/タイトルに戻る を選択する

## コンテンツ（必須）
- 表示要素:
  - タイトル: "PAUSED" の文字
  - プレイ時間: 現在のプレイ時間表示
- 操作要素:
  - 再開ボタン: ボタン → ポーズ解除してゲームに戻る
  - 設定ボタン: ボタン → 設定画面をスタックで開く
  - タイトルに戻る: ボタン → 確認ダイアログ後にタイトルシーンへ

## 操作（必須）
- 入力方式: 両対応
- フォーカスナビゲーション: 上下選択（パッド対応）

## 見た目（推奨）
- カラースキーム: 半透明オーバーレイ
- アクセントカラー: #4FC3F7（水色）
- 背景: 半透明黒（ゲーム画面がうっすら見える）
- 密度: ゆったり

## アニメーション（推奨）
- レベル: 標準
- 具体的な要件:
  - 画面表示時にオーバーレイがフェードイン（0.2s）
  - ボタンにホバー時スケールアップ（1.05x）

## 画面遷移（推奨）
- この画面への入り方: ESCキー / Startボタン
- この画面からの出方: 再開ボタン or ESC再押下
- 戻り先: ゲーム画面（TimeScale復帰）

## データバインディング（該当する場合）
- プレイ時間ラベル ← GameManager.Data.PlayTime

## 参考（任意）
- 参考ゲーム/画像: Celeste風のシンプルなポーズ画面
- 備考: MenuStackManagerに統合すること
```

## 必須項目チェックリスト

Skillは以下が未記入の場合、生成に進まず対話で確認する:

| 項目 | 未記入時の質問 |
|------|---------------|
| 画面種別 | 「どんな種類の画面ですか？ HUD/メニュー/ダイアログ/インベントリ/設定/その他」 |
| 目的 | 「この画面で何ができるようにしたいですか？」 |
| 表示要素 | 「画面に表示する情報を教えてください」 |
| 操作要素 | 「ユーザーが操作できる要素はありますか？（ボタン、スライダー等）」 |
| 入力方式 | 「操作方法は？ マウス+キーボード / ゲームパッド / 両対応」 |
